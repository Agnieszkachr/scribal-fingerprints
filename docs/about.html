About This Project

Project Overview

Scribal Fingerprints is a computational research project that investigates whether individual manuscript witnesses to the Greek New Testament carry statistically distinguishable "change profiles" — patterns of textual variation that function as quantifiable fingerprints of scribal behaviour.

The project focuses on the Gospel of John (all 21 chapters) as its primary corpus, analysing five major Greek witnesses: the papyrus P66 (c. 200 CE), Codex Sinaiticus (ℵ, 4th century), Codex Alexandrinus (A, 5th century), Codex Vaticanus (B, 4th century), and Codex Bezae (D, 5th century).

Rather than studying individual variant readings in isolation, this project constructs aggregate profiles that characterise each manuscript's overall pattern of divergence from a shared baseline. These profiles are built from contextual neural embeddings (BERT) of each variant within its local textual context, enabling a semantically informed representation of scribal change.

Data Sources

All textual data is derived from the TEI XML transcriptions provided by the International Greek New Testament Project (ITSEE) at the University of Birmingham. These transcriptions represent scholarly diplomatic editions of each manuscript.

Variant extraction uses word-level alignment via the CollateX algorithm, followed by automated classification into five typological categories. Contextual embeddings are generated using BERT (base, multilingual, cased), which provides adequate coverage of polytonic Greek orthography.

Researcher

Agnieszka B. Ziemińska is a researcher in Digital Humanities and New Testament textual criticism, applying computational methods to long-standing philological questions about scribal practice and manuscript transmission.

Code & Reproducibility

The complete analysis pipeline is open-source and available on GitHub. All results presented on this website can be reproduced by running the six Python scripts in sequence as described in the Methodology section.

This website is a static HTML/CSS/JS application generated from the pipeline outputs. It requires no server-side processing and can be served directly from the docs/ directory via GitHub Pages.
